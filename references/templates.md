# Templates & Quality Standards

每个 `.project-memory/` 文件的模板和质量标准。

---

## index.md（≤ 150 行）

入口摘要。每次对话自动加载。只放"每次都需要知道"的信息。

```markdown
# [Project Name]

[一句话：这是什么 + 用什么技术栈 + 怎么部署]

## Module Map (简版)
[顶层目录，每个一句话职责]
调用方向: A → B → C（不可反向）

## Must-Know Rules
1. [违反会导致 bug 或 review 打回的规则]
2. [...]
3. [最多 5 条]

## Testing
[必跑命令]
[条件命令]

## Danger Zones (top 5)
[文件] — [为什么危险，一句话]

## Gotchas
[坑1：你会这样做，但其实应该那样做]
[坑2：...]

## Quick Locate
| 要做什么 | 看这里 |
|---------|--------|
| [任务类型] | [文件路径] |

## Deep Docs
- 完整模块地图: @.project-memory/map.md
- 编码约定: @.project-memory/conventions.md
- 危险区域详解: @.project-memory/danger-zones.md
- 任务进度: @.project-memory/progress.md
```

### 质量标准

| 项目 | 必须满足 | 不合格示例 |
|------|---------|-----------|
| Module Map | 每个顶层目录都有职责说明 | 只列了目录名没有说明 |
| Must-Know Rules | 每条是"违反了会怎样"而非"建议怎样" | "建议用 structlog" |
| Danger Zones | 每条有文件路径 + 为什么危险 | "数据库相关的要小心" |
| Quick Locate | 每条有具体文件路径 | "在 services 目录下找" |

## map.md

完整的模块地图，包含依赖关系和任务→文件映射。

```markdown
# Module Map

## 目录结构与职责

[带 2-3 层深度的目录树，每个重要目录标注职责]

## 调用关系

[A → B → C 的调用方向图]
禁止：[列出不允许的调用方向]

## 高频修改文件 (git 热点)

| 文件 | 最近3月修改次数 | 通常改的原因 |
|------|----------------|-------------|
| ... | ... | ... |

## 任务→文件映射

| 要做什么 | 第一步看 | 相关文件 | 测试在 |
|----------|---------|---------|--------|
| ... | ... | ... | ... |

## 模块所有权（如适用）

| 模块 | 负责人/团队 | 沟通方式 |
|------|------------|---------|
| ... | ... | ... |
```

### 质量标准

- 任务→文件映射至少覆盖 5 种常见任务类型
- 每个映射包含"测试在哪里"
- 高频修改文件来自实际 git 数据，不是猜测

## conventions.md

编码约定，用"正确/错误对比对"表达。

```markdown
# Coding Conventions

## [约定类别 1：错误处理]

✓ 正确:
```[语言]
# [文件路径:行号] — 示例
[实际正确代码]
```

✗ 错误:
```[语言]
[常见错误做法]
```

为什么: [一句话解释原因]

---

## [约定类别 2：数据库访问]
[同样格式]
```

### 质量标准

| 项目 | 必须满足 |
|------|---------|
| 每条约定 | 有正确示例 + 错误示例 + 为什么 |
| 正确示例 | 引用实际项目代码（文件路径:行号） |
| 错误示例 | 是 agent 真的可能会犯的错，不是极端情况 |
| 为什么 | 解释后果（"会导致..."），不是偏好（"我们更喜欢..."） |

## danger-zones.md

高风险区域详解。每个条目必须是可操作的。

```markdown
# Danger Zones

## [文件路径]

- **入度**: [N] 个文件依赖此模块
- **依赖方**: [列出主要依赖方目录/文件]
- **为什么危险**: [具体解释]
- **改之前做**: [具体的检查步骤/命令]
- **历史事故**: [如果有，描述之前因为改这里导致的问题]

---

## [下一个文件]
[同样格式]
```

### 质量标准

| 项目 | 必须满足 | 不合格 |
|------|---------|--------|
| 入度 | 来自实际 import 分析 | 凭感觉说"很多地方用" |
| 检查命令 | 可直接执行的搜索命令 | "检查一下影响范围" |
| 为什么危险 | 说明改错的具体后果 | "这个很重要" |
